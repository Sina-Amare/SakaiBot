## Documentation Standards

All implementation documentation follows these standards:

- **Markdown format** (.md files)
- **Clear section headers** with descriptive names
- **Code examples** with syntax highlighting
- **Diagrams** in text/ASCII format where applicable
- **Cross-references** to related documentation
- **Version information** where relevant

## Contributing

When adding new features, create corresponding documentation in the appropriate subdirectories:
- `features/` - User-facing documentation
- `architecture/` - System design documentation
- `implementation/` - Implementation details
- `changelog/` - Change history
